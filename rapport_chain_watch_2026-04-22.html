Buggar

Förexisterande kodfel (identifierade och fixade)

FilStatusBeskrivning
chainwatch.py — event_type FIXAD Brandväggsevents hade inkonsekvent namngivning (ufw_block, ufw_allow). Döpt om till fw_block / fw_allow i alla parsers och tester för att reflektera att verktyget stöder flera brandväggar.
chainwatch.py — portscan-tröskel FIXAD En ensam blockerad anslutning triggade portscan_then_login. Fixat med krav på ≥ 2 distinkta destinationsportar (distinct_ports) för att filtrera ut normalt bakgrundsbrus.
chainwatch.py — lateral_movement sudo FIXAD lateral_movement kollade bara auditd execve-events. Lagt till sudo_usage-events från auth.log med korrekt svårighetsgrad: critical vid sudo till root, high vid sudo till annan användare.

Buggar hittade vid kodgranskning

FilStatusBeskrivning
chainwatch.py — IPv6 i _NETFILTER_SRC FIXAD Regex _NETFILTER_SRC matchade inte versaler i IPv6-adresser (FE80::1). Lade till A-F i teckenklassen. Två nya tester: test_ufw_block_ipv6_uppercase och test_ufw_block_ipv6_lowercase.
chainwatch.py — --since/--until-ordning FIXAD Om --since var senare än --until gav verktyget inga fel och returnerade tomma resultat i tysthet. Lagt till validering med tydligt felmeddelande och sys.exit(1).
chainwatch.py — minnesläcka i follow-läge FIXAD seen_incidents-setet i _follow_mode() växte obegränsat vid långvarig körning. Fixat med rensning varje iteration: behåll bara incidenter nyare än now - 2 × window.

Timestamp-bugg: år vid årsskifte

FilStatusBeskrivning
chainwatch.py — _parse_timestamp FIXAD Första försöket använde if dt > today: vilket rollbackade tidsstämplar med klockslag senare än nuet (t.ex. 23:59:59 när klockan är 12:00) till föregående år. Fixat med if dt.date() > today.date(): för korrekt jämförelse enbart på datumdelen.

Nya funktioner

--since / --until — tidsfiltrering

KomponentStatusDetalj
_parse_time_arg() KLAR Parsar fyra format: HH:MM, HH:MM:SS, YYYY-MM-DD HH:MM, YYYY-MM-DD HH:MM:SS. Returnerar datetime-objekt, None vid ogiltigt format.
_filter_events() KLAR Filtrerar en lista events med valfria since/until-gränser. Events utan tidsstämpel passerar alltid igenom.
CLI-flaggor KLAR --since TIME och --until TIME lagda till i argparse. Ordningsvalidering: om since > until avslutas programmet med felmeddelande.

--follow — live-läge med loggrotation

KomponentStatusDetalj
_print_follow_incident() KLAR Skriver ut en enstaka incident i realtid med tidsstämpel, kedjtyp, svårighetsgrad och käll-IP.
_follow_mode() KLAR Pollar alla tre loggkällor med konfigurerbart intervall (standard 5 s). Håller filposition per källa och läser enbart nya rader varje iteration. Stödjer Ctrl+C för ren avslutning.
Loggrotation KLAR Detekterar rotation via stat().st_size < prev_size. Vid rotation återställs positionen till 0 och hela den nya filen läses.
Minnesbegränsning KLAR seen_incidents-setet rensas varje iteration. Incidenter äldre än 2 × window tas bort för att förhindra obegränsad tillväxt.
CLI-flaggor KLAR --follow aktiverar live-läge. --interval SECONDS styr pollintervall (standard: 5 s).

Utökning av befintliga testfiler

FilStatusTillagda tester
tests/test_correlate.py UPPDATERAD 7 nya tester: enporten blockeras ej som scan, tvåportar triggar scan, sudo-till-root är critical, sudo-till-icke-root är high, sudo utanför fönster länkas ej, sudo för fel användare länkas ej, sudo och execve kombinerat.
tests/test_parse_firewall_log.py UPPDATERAD 2 nya tester: IPv6 med versaler, IPv6 med gemener. Alla ufw_block/ufw_allow uppdaterade till fw_block/fw_allow.
Obs: Follow-mode-tester använde initialt trådar med riktiga sleeps vilket gav race conditions. Ersatt med en _controlled_follow()-hjälpare som exekverar en åtgärdslista deterministiskt vid varje time.sleep-anrop — reproducerbara tester utan tidsberoenden.
